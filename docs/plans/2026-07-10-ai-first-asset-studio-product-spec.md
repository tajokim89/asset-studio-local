# AI-First Pixel Asset Studio 제품 구조 기획서

- 작성일: 2026-07-10
- 프로젝트: `/Users/tajokim/asset-studio-local`
- 상태: 확정 기획 기록, 아직 본 기획에 따른 구현은 시작하지 않음
- 기준: AI 제작이 주기능이며 수동 편집은 생성 결과를 보정하는 보조 기능

## 1. 제품 정의

Pixel Asset Studio는 범용 포토샵/Canva 클론이 아니다. 핵심 제품은 다음 흐름을 한 페이지 안에서 제공하는 **AI 기반 게임 에셋 제작 도구**다.

```text
에셋 분류 선택
→ 프로젝트 스타일과 제작 조건 선택
→ AI 생성
→ 후보 비교
→ 캔버스에 채택
→ 필요한 부분만 AI 또는 수동 수정
→ 분류별 QA
→ 게임용 PNG/시트/ZIP 출력
```

브러시, 펜슬, 지우개, 크롭, 텍스트, 도형은 중요하지만 제품의 첫인상이나 기본 진입점이 아니다. 이 도구들은 AI 생성 결과의 후처리를 위한 `수정 모드` 안에 둔다.

## 2. 확정 UX 원칙

1. 앱 시작 기본값은 `AI 모드`다.
2. `선택`, `이동`, `영역`은 두 모드에서 항상 고정 노출한다.
3. 나머지 도구는 `AI 모드`와 `수정 모드`에 따라 교체한다.
4. AI 설정은 긴 단일 폼이 아니라 에셋 대분류에 따라 필요한 옵션만 노출한다.
5. 스프라이트 설정이 UI, 타일, 오브젝트 요청에 섞이지 않아야 한다.
6. 생성 결과는 즉시 원본을 덮지 않고 결과 트레이에서 비교·채택한다.
7. 프로젝트 스타일 프로필은 모든 에셋 분류가 공유한다.
8. 수동 편집 기능은 제거하지 않고 보조 기능으로 재배치한다.
9. 기존 생성·편집 API와 현재 작업 결과를 보존하면서 단계적으로 전환한다.
10. 각 Phase는 구현, 테스트, 외부 URL 검증 후 사용자 확인을 받고 다음 Phase로 넘어간다.

## 3. 화면 정보 구조

```text
┌──────────────────────────────────────────────────────────────────┐
│ 로고 / 프로젝트       [ AI 모드 | 수정 모드 ]      저장 / 내보내기 │
├──────┬────────────────┬────────────────────────┬─────────────────┤
│ 공통 │ 모드별 설정     │       메인 캔버스       │ 속성 / 레이어     │
│ 도구 │                │                        │ 내보내기 / 기록   │
├──────┴────────────────┴────────────────────────┴─────────────────┤
│                   AI 생성 결과 트레이                            │
└──────────────────────────────────────────────────────────────────┘
```

### 3.1 상단 바

- 프로젝트명 또는 현재 프로젝트
- 중앙의 큰 모드 전환: `AI 모드 | 수정 모드`
- 캔버스 크기는 간결한 프리셋과 사용자 지정 값으로 유지
- Undo/Redo
- 사진 넣기
- 프로젝트 저장/불러오기
- 내보내기
- 가로 스크롤 동작은 유지하되 스크롤바는 보이지 않게 유지

### 3.2 고정 공통 도구

- `선택`: 레이어 및 오브젝트 선택, 변형 핸들
- `이동`: 캔버스 팬
- `영역`: 사각형, 원형, 올가미, 브러시 영역

`영역`은 AI 모드에서는 재생성/교체 범위로, 수정 모드에서는 복사/잘라내기/마스크/크롭 보조 범위로 사용한다.

### 3.3 오른쪽 패널

현재처럼 모든 기능을 세로로 쌓지 않고 다음 탭으로 정리한다.

- `속성`: X/Y, W/H, 회전, 불투명도, 반전, 정렬, 피벗
- `레이어`: 이름, 순서, 숨김, 잠금, 그룹, 복제, 삭제
- `내보내기`: 선택한 에셋 분류에 맞는 출력
- 히스토리는 오른쪽 하단의 접이식 영역으로 둔다.

## 4. AI 모드

AI 모드는 다음 두 하위 흐름으로 구성한다.

```text
[새 에셋 제작] [선택 영역 AI 수정]
```

### 4.1 새 에셋 제작 공통 순서

1. 에셋 대분류
2. 하위 종류
3. 사용자가 입력하는 핵심 제작 요청
4. 프로젝트 스타일
5. 분류별 상세 설정
6. 출력 크기와 배경
7. 큰 `AI 에셋 생성` 버튼
8. 접힌 `고급 설정`

고급 설정에만 배치할 항목:

- 자동 조립된 전체 프롬프트
- Negative Prompt
- 모델/품질
- 세부 배경 처리
- 원시 API 값
- 디버그 정보

## 6. 선택 영역 AI 수정

사용자는 공통 `선택`과 `영역` 도구로 대상 이미지 일부를 지정한다.

필수 정보:

- 대상 레이어
- 영역 종류와 bbox
- 수정 요청
- Negative
- 적용 방식

빠른 작업:

- 오브젝트 교체
- 색상/재질 변경
- 손상/오염 추가
- 스타일 맞추기
- 배경 제거
- 투명 PNG 정리
- 선택 영역 재생성
- 변형 만들기

적용 방식:

- 새 패치 레이어
- 원본 교체
- 새 변형
- 다시 생성
- 취소

원본 교체도 원본 레이어를 삭제하지 않고 숨김 보존한다.

## 7. 수정 모드

AI 결과를 수동으로 보정하는 보조 모드다.

도구:

- 크롭
- 브러시
- 펜슬
- 지우개
- 마스크
- 텍스트
- 도형
- 정렬

각 도구를 선택했을 때 해당 옵션만 왼쪽 패널에 표시한다. AI 생성 설정은 수정 모드에서 숨기고 `AI 모드로 돌아가기`를 명확하게 제공한다.

## 8. AI 결과 트레이

생성 결과는 캔버스에 자동 덮어쓰지 않고 하단 트레이에 쌓는다.

필수 기능:

- 후보 썸네일 비교
- 선택 후보 확대
- 캔버스에 추가
- 새 변형
- 선택 영역 수정으로 전달
- 삭제
- 실패 후보 분리
- 이전 생성 결과 복구

결과와 캔버스 레이어는 서로 다른 상태로 관리한다. 승인한 후보만 캔버스와 프로젝트 자산에 들어간다.

## 9. 프로젝트 스타일 프로필

프로젝트 전체의 통일감을 고정하는 공유 설정이다.

필드:

- 프로필 이름
- 픽셀 밀도
- 팔레트
- 주재료/보조 재료
- 외곽선
- 명암 단계
- 광원 방향
- 카메라/시점
- 금지 요소
- 기준 이미지
- 사용자 스타일 메모

스타일 프로필은 프로젝트 JSON에 저장하고 스프라이트, 타일, UI, 오브젝트 요청에 공통 적용한다.

## 10. 권장 API 계약

공통 필드:

```json
{
  "asset_family": "ui",
  "asset_type": "button",
  "style_profile": "dungeon_cleanup",
  "prompt": "wide work-order confirmation button",
  "negative": "text, logo, watermark",
  "output": {
    "width": 320,
    "height": 96,
    "transparent": true,
    "padding": 12
  }
}
```

스프라이트 전용:

```json
{
  "sprite": {
    "animation_mode": "walk4",
    "direction_mode": "single",
    "target_direction": "S",
    "reference_direction": "S",
    "frame_count": 4
  }
}
```

UI 전용:

```json
{
  "ui": {
    "nine_slice": true,
    "nine_slice_margin": 16,
    "content_safe_area": 24,
    "states": ["normal", "hover", "pressed"],
    "text_free": true
  }
}
```

타일 전용:

```json
{
  "tile": {
    "tile_size": 32,
    "layout": "tileset",
    "seamless": true,
    "connections": ["N", "E", "S", "W"]
  }
}
```

오브젝트 전용:

```json
{
  "object": {
    "view": "three_quarter",
    "pivot": "bottom_center",
    "world_scale": 64,
    "variants": 1,
    "shadow": false
  }
}
```

잘못된 분류의 필드는 전송하지 않는다. 예를 들어 UI 요청에 `animation_mode`, `direction_mode`, 스프라이트 잔재 정리 옵션이 들어가면 안 된다.

## 14. 의사결정 기록

- 사용자는 에셋을 직접 페이지에서 제작한다. 에이전트의 역할은 페이지와 워크플로를 잘 세팅하는 것이다.
- AI 생성이 메인이고 수동 편집은 부가기능이다.
- `선택`, `이동`, `영역`은 항상 고정한다.
